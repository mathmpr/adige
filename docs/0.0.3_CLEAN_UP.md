# CLEAN UP 0.0.3

This document tracks the cleanup line that follows the `0.0.1` core stabilization and the `0.0.2` ORM stabilization.

Scope of this cleanup:
- remove dead or deprecated framework structures that are no longer part of the stabilized execution flow
- preserve useful behavior when it still fits the current architecture
- reduce duplicated discovery/reflection logic
- keep console execution aligned with the current controller/router contracts

Out of scope for now:
- new console features outside the current controller-based execution model
- unrelated cleanup across web, middleware or ORM modules
- replacing the current console output helper

Architectural decision captured in this step:
- console autodiscover remains based on the stabilized router contract in [`src/core/routing/Router.php`](/home/mathmpr/PhpstormProjects/adige/src/core/routing/Router.php)
- command suggestions do not restore the old static command registry
- instead, suggestions are derived from the same controller namespace contract used by autodiscover
- command enumeration prefers Composer autoload metadata and uses PSR-4 namespace resolution as fallback

Why this direction was chosen:
- the stabilized console path already runs through `ConsoleRequest -> Router -> controller action`
- bringing `didYouSay()` back through the old `Console`/`Command` model would reintroduce a second command system
- the new catalog preserves the good UX part of the old implementation without reopening the deprecated execution model
